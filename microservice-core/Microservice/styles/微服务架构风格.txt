1. 分层式架构风格 Layer Architect
   分层架构将软件元素按照layer的方式进行组织，每层定义明确的职责
   // 0. View UI层
   // 1. Controller 控制层
   // 2. Service 业务层
   // 3. ORM层/Dao数据持久层
   // 4. 数据库系统
   - 层之间有明确的依赖关系，每层都依赖其下方的层
   - 在缺少数据持久层的情况下很难测试业务逻辑层
   - 单一的数据持久化层如何与不同的DB进行交互？

2. 六边形架构风格 Hexagon Architect
   分层架构的替代品，能够解决分层架构的一些弊端
